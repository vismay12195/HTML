Define the terms: Website, Webpage, Web browser, Web server, HTML, CSS


  • Website:A collection of related web pages and multimedia content accessible through a unique domain name or URL.
  • Webpage:A single document or file that forms part of a website and is displayed in a web browser.
  • Web server:A computer or system that hosts and delivers web content to clients upon request, responding to browser requests and sending back the requested files or data.
  • Web browser:Software application that allows users to access, view, and interact with websites and web pages on the internet by interpreting HTML, CSS, and JavaScript code.
  • HTML:Hypertext Markup Language; the standard markup language used to structure and present content on the web, defining elements and components of a webpage.
  • CSS: Cascading Style Sheets; a stylesheet language used to describe the presentation and formatting of HTML documents, controlling the visual aspects and layout of webpages.

Create link-pseudo classes using external css, to format links on the pages.


To create link pseudo-classes using external CSS, we can use the following selectors and properties:

For ex:
  a:link {
    color: blue;
    text-decoration: underline;
  }
   a:visited {
    color: purple;
  }
   a:hover {
    color: red;
    text-decoration: none;
  }
   a:active {
    color: green;
  }

By combining these pseudo-classes with the "a" selector (which selects all anchor tags), we can apply different styles to links based on their state.

To apply these styles externally, create a separate CSS file (e.g., "styles.css") and link it in the HTML document's <head> section using the <link> tag.


Create a dynamic pseudo class using HTML, CSS


In HTML and CSS, we cannot create custom dynamic pseudo-classes. Pseudo-classes are predefined in CSS and are based on certain states or conditions of elements, such as :hover, :active, :visited, etc.

However, you can use JavaScript to add or remove classes dynamically based on certain events or conditions, which can achieve similar effects.

For ex:
  <button id="myButton">Click me<⁄button> //HTML code

  button.dynamic-class {
    background-color: yellow;
  } //CSS code

  //JavaScript code
  var button = document.getElementById('myButton');
  button.addEventListener('click', function(){
  button.classicList.add('dynamic-class');});


In this example, when the button is clicked, the JavaScript code adds the "dynamic-class" to the button element using the classList.add() method. The CSS rule .dynamic-class is defined with a yellow background color.

By dynamically adding or removing classes on elements using JavaScript, we can achieve dynamic styling effects that appear similar to pseudo-classes.


What are the new tags added in HTML5?


HTML5 introduced several new tags to enhance the structure and semantics of web documents. Here are some of the notable new tags added in HTML5:

  • <header>: Represents the introductory content or a group of navigational elements for a section or a page.
  • <nav>: Represents a section of a page that contains navigation links.
  • <main>: Represents the main content of a document. There should be only one
    element per page.
  • <article>: Represents a self-contained composition within a document, such as a blog post, a news story, or a forum post.
  • <section>: Represents a thematic grouping of content, typically with a heading.
  • <aside>: Represents content that is tangentially related to the main content, such as sidebars, pull quotes, or advertisements.
  • <figure> and <figcaption>: <figure> is used to encapsulate self-contained content, such as images, diagrams, illustrations, or code examples. <figcaption> is used to provide a caption or description for the content inside the <figure> element.
  • <footer>: Represents the footer section of a document or a section. It typically contains information about the author, copyright notice, links to related documents, or contact information.
  • <video> and <audio>: <video> is used to embed video content in a document, and <audio> is used to embed audio content.
  • <datalist> and <option>: <datalist> provides a pre-defined list of options for an input field, and <option> represents an option within a <select> element or a <datalist> element.

How to embed audio and video in a webpage?


Embedding Audio:

  1. Provide a source for the audio file using the <source> tag within the <audio> tag.
  2. Specify the audio file's format using the "src" attribute in the <source> tag.
  3. Add fallback content between the opening and closing <audio> tags for browsers that don't support the <audio> element.

For ex:
  <audio controls>
    <source src="audio-file.mp3" type="audio/mp3">
    <source src="audio-file.ogg" type="audio/ogg">
    Your browser does not support the audio element.
  <⁄audio>

  1. Provide a source for the video file using the <source> tag within the <video> tag.
  2. Specify the video file's format using the "src" attribute in the <source> tag.

For ex:
  <video controls>
    <source src="video-file.mp4" type="video/mp4">
    <source src="video-file.webm" type="video/webm">

Canvas and SVG tags


Both the <canvas> and <svg> tags in HTML5 are used for drawing graphics, but they have different approaches and use cases.

<canvas>:

  • The <canvas> element provides a drawing surface on which you can render dynamic graphics using JavaScript.
  • It acts as a bitmap or raster graphics container, allowing you to manipulate pixels directly.
  • You can use JavaScript to draw shapes, images, and animations on the canvas.
  • The content drawn on the canvas is not part of the document's DOM, so it's not directly accessible or searchable.
  • It is suitable for complex, dynamic, and interactive graphics, such as games or data visualizations.

For ex:
  <canvas id="myCanvas" width="400" height="200"><⁄canvas>

<svg>:

  • Scalable Vector Graphics element defines a vector-based graphics container.
  • It uses XML markup to describe graphical objects and elements, allowing for resolution-independent graphics.
  • <svg> elements are part of the document's DOM and can be styled, animated, and interacted with using CSS and JavaScript.
  • We can draw shapes, paths, text, and images using SVG-specific elements and attributes.
  • The content within an <svg> element is accessible, searchable, and can be styled or manipulated using CSS and JavaScript.

For ex:
  <svg width="400" height="200">
    <rect x="50" y="50" width="300" height="100" fill="blue">
    <circle cx="200" cy="150" r="50" fill="red">
  <⁄svg>

Use <canvas> for complex, dynamic graphics, and <svg> for scalable, resolution-independent, and accessible graphics.
